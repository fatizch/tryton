Architecture
============

Ce document cherche à permettre l'acquisition d'une compétence minimum sur les
aspects métiers de **Coog** (qu'est-ce qu'un contrat, un risque. Qu'est-ce
qu'une garantie, quels sont les éléments comptables liés à la vie d'un contrat
d'assurance, etc.)

En outre, ces éléments « métier » seront accompagnés d'informations techniques
permettant aux développeurs de prendre en main le modèle, ainsi que les outils
techniques utilisés dans le cadre du développement de **Coog**.

Le *glossaire* contient des correspondances entre les noms de modèles
« techniques » et les objets métiers, les *généralités* des informations
générales sur des modèles transverses, ou sur comment est développé **Coog** de
façon générale.
